TOLEGO · PROPUESTA E2E 03 · DOLOR DESCONOCIDO

Una venta perdida no vale $30.000

El cliente objetivo de Tolego es coleccionista. Perder una venta no es perder un pedido: es perder a un cliente que habría recomprado por años.

El dolor invisible

Cada respuesta tardía o quiebre de stock no descuenta un pedido — rompe una relación de recompra.

El coleccionista vuelve por la siguiente pieza, la nueva saga, el regalo a otro fan. Eso no se recupera con descuentos.

¿Por qué es crítico?

25% de fuga hoy

~50 pedidos/mes perdidos por atención fuera de horario y quiebres de stock no detectados

Cero trazabilidad del viaje

No se sabe qué coleccionista regresa ni qué franquicia rinde — decisiones por percepción

Sin registro, sin recompra

Sin base de datos de cliente no hay lanzamiento dirigido, cross-sell ni LTV medible

Resolver esto no es automatización: es proteger el activo más valioso del negocio — la base de coleccionistas recurrentes.

TOLEGO · PROPUESTA E2E 06 · E2E · CONSUMO + ETL

Consumo Digital + ETL

Tres capas de almacenamiento especializadas y un pipeline ETL que consolida el negocio en un modelo estrella.

Relacional — Cloud SQL

Inventario, transacciones y catálogo con integridad referencial. SKUs con la codificación Tolego existente.

NoSQL — Firestore

Pedidos en tiempo real vía webhook. Escritura rápida desde Dialogflow CX.

Warehouse — BigQuery

Modelo estrella con fact_ventas y dimensiones producto, cliente, tiempo, canal.

Pipeline ETL

→ Extract

Cloud SQL · Firestore · Document AI

→ Transform

Cloud Functions: normalizar SKUs, calcular margen, deduplicar

→ Load

BigQuery cada 6 h, consumido por Looker Studio

Modelo estrella — fact_ventas

  • Producto: SKU, categoría, variante
  • Cliente: ciudad, canal, recompras
  • Tiempo: día, mes, temporada
  • Canal: Instagram / WhatsApp

TOLEGO · PROPUESTA E2E 07 · E2E · GOBERNANZA

Gobernanza y Habeas Data

Políticas de calidad, seguridad y cumplimiento alineadas con la Ley 1581 de 2012. Tolego es Responsable; Google Cloud, Encargado.

✅

Calidad

Dialogflow valida SKUs contra catálogo. Cloud Functions verifica montos y deduplica por cliente+fecha+SKU.

🔒

Seguridad

IAM mínimo privilegio, cifrado en reposo y tránsito, Firestore con reglas de webhook autorizado.

👤

Acceso

Fundador: acceso total. Consultor: temporal durante implementación. Cloud Audit Logs para trazabilidad.

🗄️

Retención

Transaccional: indefinido. Comprobantes OCR: 30 días. Supresión a petición: 10 días hábiles.

Habeas Data — Ley 1581 de 2012

Autorización

El bot solicita autorización explícita con timestamp en el primer contacto; queda vinculada al id_cliente.

Derechos ARCO

Acceso, rectificación, cancelación y oposición vía intent del bot ("ver mis datos", "borrar mis datos").

RNBD + SIC

Documentación lista en Notion; registro en SIC al superar el umbral regulatorio. Notificación de incidentes con Cloud Audit Logs.
